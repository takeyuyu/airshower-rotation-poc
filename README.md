# エアシャワー「回転」判定の最小PoC ＜第2作・後編＞

クリーンルームに入る前のエアシャワー。正しい入り方は、
**両腕を上げて、その場でゆっくり360度回転し**、全身を気流にさらすことだ。

第1作（前編）では、このうち「腕を上げているか」を判定した。
本PoC（第2作・後編）は、残り半分の **ちゃんと回ったか** を判定する。

- 前作：腕の上げの判定 → https://github.com/takeyuyu/airshower-tap-poc
- 記事（後編）：https://note.com/major_bobcat4320/n/ne1041a66ad4d?app_launch=false

## このコードが示すこと

回り方は、人によって揃わない。そこで「何度回ったか（到達角度）」だけで判定しようとすると、
壁にぶつかる。**ゆっくり一周した人も、速くクルッと回っただけの人も、到達角度は同じ360度**
だからだ。角度だけでは、一周はしたが各向きに風が当たっていない人を取り逃がす。

そこで、動作の「形」ではなく、**体の向き（角度）の「時間変化」**を信号として使う。
（噴霧PoCの「明るさ」、前編の「腕の高さ」と同じ考え方。捉えやすい1次元の波に落とす。）

骨格推定で得た左右の肩の見かけの幅などから体の向きを割り出し、累積回転角を追う。
そこから、引き算と数えるだけの素朴な特徴量を3つ出す：

- **到達角度**：最後にどこまで回ったか（一周＝360度か）
- **踏んだ向き**：正面・横・背面・逆横の4区画のうち、いくつを通ったか
- **最小滞留**：4区画のうち、いちばん滞在が短い向きの時間割合（＝最も風が当たらない向き）

これで、3つの状態を切り分けられるかを合成データで確認する：

- **良好**：一周し、各向きに十分とどまっている
- **形だけ**：一周はしたが、速すぎて各向きの滞留が足りない
- **未実施**：そもそも一周していない（半周で止まる等）

そして、**到達角度では同じ360度に見える2人が、「滞留」を足すと
くっきり分かれる**ことを示す。前編の「回数では分けられない／高さと保持で分かれる」と
対称の構造になっている。

## 前提と位置づけ

- **実データは一切使用していません。** すべてプログラムで生成した合成データです。
- 体の向きは、骨格推定で得られる座標から割り出す「代理データ」として合成しています。
- **回転は背面で骨格推定が乱れる**ため、実機ではここで扱うほど素直な信号は得にくい点に注意。
  この「いちばん見たい背中側がいちばん見えない」難しさは、記事本文でも述べています。
- ISO 14644 などの規格は、結果としての清浄度（粒子数）を規定しますが、エアシャワー内で
  人がどう動くべきかは規定していません。動作は各現場の手順任せです。
- これは「原理が成立するか」を確かめる第一歩であり、実機での動作を保証するものではありません。

## 実行方法

```
pip install numpy matplotlib
python airshower_rotation_poc.py
```

実行すると、判定結果がコンソールに表示され、同じフォルダに2枚の図が出力されます。

- `rotation_waves.png` … 3人ぶんの体の向きの時間波形（良好／形だけ／未実施）

## 判定ロジックの要点

到達角度だけに頼らないことが肝です。

```
到達角度が小さい（< しきい値）              → 未実施（一周していない）
一周はしたが、最小滞留が短い（< しきい値）   → 形だけ（速すぎる）
一周し、各向きに十分とどまっている          → 良好
```

「一周しても中身は違う」を、角度と滞留の2軸で切り分ける。
前編と同じく、たいそうなモデルは使っていません。
